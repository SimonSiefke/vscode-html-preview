      <td class="d-block comment-body markdown-body  js-comment-body">
          <h3>What problem does this feature solve?</h3>
<p>When I first started using Vue (a long time ago :) I've liked how simple it was and the ability to debug any object value by printing it in the template using <code>{{ object }}</code>. I've kept this reflex when prototyping because it is so handy and often times quicker than waiting for devtools to load.<br>
Now that we can use reactive Map/Set objects, I recently played with vue-next and reactive Maps and realized this was not working like regular objects. I wish we could do the same kind of thing.</p>
<h3>What does the proposed API look like?</h3>
<p>when evaluating an object in the template, vue is stringifying the object behind the scenes.<br>
Likewise we could have an 'universal' JSON stringify replacer that handles Map and Set objects.</p>
<p>for instance (snippet taken somewhere on stackoverflow):</p>
<div class="highlight highlight-source-js"><pre><span class="pl-c1">JSON</span>.<span class="pl-c1">stringify</span>(map, <span class="pl-k">function</span> <span class="pl-en">replacer</span> (<span class="pl-smi">key</span>, <span class="pl-smi">value</span>) {
      <span class="pl-k">const</span> <span class="pl-c1">originalObject</span> <span class="pl-k">=</span> <span class="pl-c1">this</span>[key]
      <span class="pl-k">if</span> (originalObject <span class="pl-k">instanceof</span> <span class="pl-c1">Map</span>) {
        <span class="pl-k">return</span> {
          dataType<span class="pl-k">:</span> <span class="pl-s"><span class="pl-pds">'</span>Map<span class="pl-pds">'</span></span>,
          value<span class="pl-k">:</span> <span class="pl-c1">Array</span>.<span class="pl-en">from</span>(<span class="pl-smi">originalObject</span>.<span class="pl-c1">entries</span>()) <span class="pl-c"><span class="pl-c">//</span> or with spread: value: [...originalObject]</span>
        }
      } <span class="pl-k">else</span> {
        <span class="pl-k">return</span> value
      }
    }, <span class="pl-c1">2</span>))</pre></div>
<p>This way it could as well recursively walk through nested reactive Map/Set objects and update accordingly.<br>
It would be a nice QoL improvement in my opinion, I dunno if it could add some overhead.<br>
In any case I'm open to submit a PR if the idea is approved</p>
