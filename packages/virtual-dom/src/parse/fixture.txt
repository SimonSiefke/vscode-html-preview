	<h1 id="firstHeading" class="firstHeading" lang="en">Conditional comment</h1>

	<div id="bodyContent" class="mw-body-content">
		<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>
		<div id="contentSub"></div>


		<div id="jump-to-nav"></div>
		<a class="mw-jump-link" href="#mw-head">Jump to navigation</a>
		<a class="mw-jump-link" href="#p-search">Jump to search</a>
		<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p><b>Conditional comments</b> are <a href="/wiki/Conditional_(computer_programming)" title="Conditional (computer programming)">conditional statements</a> interpreted by Microsoft <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> versions 5 through 9 in <a href="/wiki/HTML" title="HTML">HTML</a> source code. Conditional comments can be used to provide and hide code to and from Internet Explorer. Conditional comments are no longer supported in IE10 and IE11.
</p><p>Conditional comments in HTML<sup id="cite_ref-MS1_1-0" class="reference"><a href="#cite_note-MS1-1">&#91;1&#93;</a></sup> first appeared in <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>'s <a href="/wiki/Internet_Explorer_5" title="Internet Explorer 5">Internet Explorer 5</a> browser, although support has now been deprecated. In <a href="/wiki/Internet_Explorer_10" title="Internet Explorer 10">Internet Explorer 10</a>, HTML conditional comments are not supported when the page is in standards mode (document mode 10).<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup> JScript conditional comments were introduced in <a href="/wiki/Internet_Explorer_4" title="Internet Explorer 4">Internet Explorer 4</a> and they continue to be supported in <a href="/wiki/Internet_Explorer_10" title="Internet Explorer 10">Internet Explorer 10</a>, in standards mode or compatibility mode.
</p>

<h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Conditional_comment&amp;action=edit&amp;section=1" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Here is a simple example that demonstrates how conditional comments work.
</p>
<div class="mw-highlight mw-highlight-lang-html mw-content-ltr" dir="ltr"><pre><span></span><span class="c">&lt;!--[if IE 6]&gt;</span>
<span class="c">&lt;p&gt;You are using Internet Explorer 6.&lt;/p&gt;</span>
<span class="c">&lt;![endif]--&gt;</span>
</pre></div>
<h2><span class="mw-headline" id="Syntax">Syntax</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Conditional_comment&amp;action=edit&amp;section=2" title="Edit section: Syntax">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>There are two types of "conditional comments": <i>downlevel revealed</i>, and <i>downlevel hidden</i>.
</p><p>The basic syntax of each type of comment is shown in the following table. The first comment shown is the basic HTML Comment, which is included for the purpose of comparison and to illustrate the different syntax used by each type of conditional comment.
</p>
<table class="wikitable">
<tbody><tr>
<th>Comment type
</th>
<th>Syntax or possible value
</th></tr>
<tr>
<td>standard HTML comment
</td>
<td>&lt;!--&#160;Comment content &#160;--&gt;
</td></tr>
<tr>
<td>downlevel-hidden
</td>
<td>&lt;!--[if <i>expression</i>]&gt; <i>HTML</i> &lt;![endif]--&gt;
</td></tr>
<tr>
<td>downlevel-revealed
</td>
<td>&lt;![if <i>expression</i>]&gt; <i>HTML</i> &lt;![endif]&gt;
</td></tr></tbody></table>
<p>The <i>HTML</i> shown inside the syntax block in each of the conditional comments denotes any block of HTML content, including script. Both types of conditional comment use a conditional <i>expression</i> to indicate whether the content inside the comment block should be parsed or ignored. The conditional expression is formed from a combination of feature, operator, and/or value, depending on the feature.  The following table shows the supported features and describes the values each feature supports.
</p>

<tbody><tr>
<th>Item
</th>
<th>Example
</th>
<th>Comment

<tr>

<td>[if IE]
</td>
<td>The string "IE" is a <i>feature</i> corresponding to the version of Internet Explorer used to view the Web page.
</td></tr>
<tr>
<td>value
</td>
<td>[if IE 7]
</td>
<td>An integer or floating point numeral corresponding to the <i>version</i> of the browser. Returns a Boolean value of true if the version number matches the browser version. For more information, see <a href="/wiki/Version_vector" title="Version vector">Version vectors</a>.
</td></tr>
<tr>
<td>WindowsEdition
</td>
<td>[if WindowsEdition]
</td>

</td></tr>
<tr>
<td>value
</td>
<td>[if WindowsEdition 1]
</td>
<td>An integer corresponding to the <i>edition</i> of Windows used to view the Web page. Returns a Boolean value of true if the value matches the edition being used. For information about supported values and the editions they describe, see the <i>pdwReturnedProductType</i> parameter of the GetProductInfo function.
</td></tr>

<h3><span class="mw-headline" id="Downlevel-hidden_conditional_comment">Downlevel-hidden conditional comment</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Conditional_comment&amp;action=edit&amp;section=3" title="Edit section: Downlevel-hidden conditional comment">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Below are two examples of a "downlevel hidden" conditional comment:
</p>
<div class="mw-highlight mw-highlight-lang-html mw-content-ltr" dir="ltr"><pre><span></span><span class="c">&lt;!--[if IE 8]&gt;</span>
<span class="c">&lt;link href=&quot;ie8only.css&quot; rel=&quot;stylesheet&quot;&gt;</span>
<span class="c">&lt;![endif]--&gt;</span>
</pre></div>
<p>or
</p>
<div class="mw-highlight mw-highlight-lang-html mw-content-ltr" dir="ltr"><pre><span></span><span class="c">&lt;!--[if lte IE 7]&gt;</span>
<span class="c">&lt;style type=&quot;text/css&quot;&gt;</span>
<span class="c">/* CSS here */</span>
<span class="c">&lt;/style&gt;</span>
<span class="c">&lt;![endif]--&gt;</span>
</pre></div>
<p><br />
The directive in the first example will let IE 8 read the specified CSS file, while IE 7 or older IE versions will ignore it. Browsers other than IE will also ignore it because it looks like a standard HTML comment. The tag in the second example will let IE versions 5.0 through 7 read the internal CSS style. With different uses of this tag you can also single out IE 6, IE 5, or versions of IE that are newer (greater) or older (less) than a specified version.
</p>
<h3><span class="mw-headline" id="Downlevel-revealed_conditional_comment">Downlevel-revealed conditional comment</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Conditional_comment&amp;action=edit&amp;section=4" title="Edit section: Downlevel-revealed conditional comment">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Below is an example of a "downlevel revealed" conditional 'comment', which is <i>not an (X)HTML</i> comment at all, despite the misleading name,  using the default Microsoft syntax:
</p>
<div class="mw-highlight mw-highlight-lang-html mw-content-ltr" dir="ltr"><pre><span></span><span class="cp">&lt;![if !IE]&gt;</span>
<span class="p">&lt;</span><span class="nt">link</span> <span class="na">href</span><span class="o">=</span><span class="s">&quot;non-ie.css&quot;</span> <span class="na">rel</span><span class="o">=</span><span class="s">&quot;stylesheet&quot;</span><span class="p">&gt;</span>
<span class="cp">&lt;![endif]&gt;</span>
</pre></div>
<p>This example shows content that should be exposed only to non-IE browsers, as the condition evaluates to "false" on IE (and hence the content is ignored), while the tags themselves are unrecognized (and hence ignored) on non-IE browsers. This is not valid HTML or XHTML.
</p><p>Microsoft acknowledges this syntax is not standardized markup,<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup> intending these tags to be overlooked by other browsers and expose the content in the middle. In order to ensure compliance with W3C standards, some web developers use an alternative technique<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup> for downlevel-revealed conditional comments:
</p>
<div class="mw-highlight mw-highlight-lang-html mw-content-ltr" dir="ltr"><pre><span></span><span class="c">&lt;!--[if !IE]&gt;--&gt;</span>
<span class="p">&lt;</span><span class="nt">link</span> <span class="na">href</span><span class="o">=</span><span class="s">&quot;non-ie.css&quot;</span> <span class="na">rel</span><span class="o">=</span><span class="s">&quot;stylesheet&quot;</span><span class="p">&gt;</span>
<span class="c">&lt;!--&lt;![endif]--&gt;</span>
</pre></div>
<p>While somewhat confusing in structure, this specific syntax is valid (X)HTML and is useful for conditional sections intended specifically for non-IE browsers; if the condition evaluates to <i>true</i> (for example, if writing code meant to display on non-IE browsers <i>and</i> on some versions of IE), IE will then display the "--&gt;" present before the HTML content.  This problem is easily solved by prepending "&lt;!" to the initial "--&gt;" as follows:
</p>
<div class="mw-highlight mw-highlight-lang-html mw-content-ltr" dir="ltr"><pre><span></span><span class="c">&lt;!--[if gt IE 6]&gt;&lt;!--&gt;</span>
This code displays on non-IE browsers and on IE 7 or higher.
<span class="c">&lt;!--&lt;![endif]--&gt;</span>
</pre></div>
<p>The extra "&lt;!" is ignored by non-IE browsers; it is also ignored by IE regardless of the condition because if <i>false</i>, everything within the conditional comment is ignored, and if <i>true</i>, the resulting tag <code>&lt;!--&gt;</code> is unrecognized and therefore ignored.
</p>
<h3><span class="mw-headline" id="Conditional_comments_in_JScript">Conditional comments in JScript</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Conditional_comment&amp;action=edit&amp;section=5" title="Edit section: Conditional comments in JScript">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<p>Starting with Internet Explorer 4, there exists a similar proprietary mechanism for adding conditional comments  within <a href="/wiki/JScript" title="JScript">JScript</a>, known as conditional compilation.<sup id="cite_ref-MS2_5-0" class="reference"><a href="#cite_note-MS2-5">&#91;5&#93;</a></sup>
</p><p>Code examples:
</p>
<div class="mw-highlight mw-highlight-lang-javascript mw-content-ltr" dir="ltr"><pre><span></span><span class="o">&lt;</span><span class="nx">script</span><span class="o">&gt;</span>
<span class="cm">/*@cc_on</span>
<span class="cm">  document.write(&quot;You are using IE4 or higher&quot;);</span>
<span class="cm">@*/</span>
<span class="o">&lt;</span><span class="err">/script&gt;</span>
</pre></div>
<p>There were also several predefined variables,<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup> though these cannot be relied on any longer as Microsoft altered the JScript engine of IE6 with XP SP3 and it now reports as:
</p>
<div class="mw-highlight mw-highlight-lang-javascript mw-content-ltr" dir="ltr"><pre><span></span><span class="err">@</span><span class="nx">_jscript_version</span> <span class="o">==</span> <span class="mf">5.7</span>
</pre></div>
<p>As a result, a possible way to detect Internet Explorer version using conditional compilation can be seen below:
</p>
<div class="mw-highlight mw-highlight-lang-javascript mw-content-ltr" dir="ltr"><pre><span></span><span class="o">&lt;</span><span class="nx">script</span><span class="o">&gt;</span>
<span class="cm">/*@cc_on</span>

<span class="cm">  @if (@_jscript_version == 11)</span>
<span class="cm">    document.write(&quot;You are using IE11 with an older document mode&quot;);</span>
<span class="cm">  @elif (@_jscript_version == 10)</span>
<span class="cm">    document.write(&quot;You are using IE10&quot;);</span>
<span class="cm">  @elif (@_jscript_version == 9)</span>
<span class="cm">    document.write(&quot;You are using IE9&quot;);</span>
<span class="cm">  @elif (@_jscript_version == 5.8)</span>
<span class="cm">    document.write(&quot;You are using IE8&quot;);</span>
<span class="cm">  @elif (@_jscript_version == 5.7)</span>
<span class="cm">    document.write(&quot;You are using IE&quot; + (!window.XMLHttpRequest ? 6 : 7));</span>
<span class="cm">  @elif (@_jscript_version == 5.6)</span>
<span class="cm">    document.write(&quot;You are using IE6&quot;);</span>
<span class="cm">  @elif (@_jscript_version == 5.5)</span>
<span class="cm">    document.write(&quot;You are using IE5.5&quot;);</span>
<span class="cm">  @elif (@_jscript_version &lt; 5.5)</span>
<span class="cm">    document.write(&quot;You are using a version older than IE5.5&quot;);</span>
<span class="cm">  @else</span>
<span class="cm">    document.write(&quot;You are using an unknown version of IE&quot;);</span>
<span class="cm">  @end</span>
<span class="cm"> </span>
<span class="cm">@*/</span>
<span class="o">&lt;</span><span class="err">/script&gt;</span>
</pre></div>
<p>However, conditional compilation is no longer supported in <a href="/wiki/Internet_Explorer_11" title="Internet Explorer 11">Internet Explorer 11</a> Standards mode.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup>
</p>
